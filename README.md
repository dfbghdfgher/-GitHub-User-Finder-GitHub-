## Описание программы

Приложение для поиска пользователей GitHub с возможностью сохранения избранных профилей в локальный JSON‑файл. Позволяет быстро находить пользователей по имени, просматривать основную информацию о них и формировать персональный список избранных аккаунтов.

### Возможности

* Поиск пользователей GitHub по имени через официальный API.
* Отображение подробной информации о найденном пользователе:
  * логин;
  * полное имя;
  * компания;
  * местоположение;
  * количество публичных репозиториев.
* Добавление пользователей в список избранного.
* Сохранение списка избранных пользователей в файл `users.json`.
* Загрузка избранных пользователей при запуске приложения.
* Валидация ввода (проверка на пустое поле и минимальную длину).
* Обработка ошибок:
  * отсутствие интернет‑соединения;
  * ошибки API (в т. ч. код 404 — пользователь не найден);
  * повреждение файла `users.json`.
* Удобный графический интерфейс с прокруткой для длинных списков.
* Поиск по нажатию клавиши Enter.

### Примеры использования программы

**Пример 1. Базовый поиск пользователя**

1. Запустите приложение командой `python main.py`.
2. В поле «Введите имя пользователя GitHub:» введите `octocat`.
3. Нажмите кнопку «Найти» или клавишу Enter.
4. В разделе «Результаты поиска» отобразится информация: "agibalova816-art | None | Компания: None | Локация: None | Репозиториев: 3
5. Результат подтверждает, что приложение успешно подключилось к GitHub API и получило данные.

**Пример 2. Поиск реального пользователя**

1. Введите в поле поиска имя реального разработчика, например `torvalds` (Линус Торвальдс).
2. Нажмите «Найти».
3. В результатах отобразится информация о пользователе, включая:
* логин: `torvalds`;
* имя: `Linus Torvalds`;
* компанию (если указана);
* локацию (если указана);
* количество публичных репозиториев.
4. Выберите строку с пользователем в списке результатов.
5. Нажмите кнопку «Добавить в избранное».
6. Пользователь появится в разделе «Избранные пользователи».

**Пример 3. Работа с несколькими пользователями**

1. Выполните поиск для нескольких пользователей подряд:
* `torvalds`;
* `defunkt` (Chris Wanstrath, сооснователь GitHub);
* `moby` (Solomon Hykes, создатель Docker).
2. После каждого успешного поиска добавляйте пользователя в избранное.
3. Проверьте раздел «Избранные пользователи» — все три профиля должны там отображаться.
4. Закройте приложение и запустите его снова.
5. Убедитесь, что список избранных сохранился в `users.json` и отображается корректно.

**Пример 4. Обработка ошибок ввода**

* **Сценарий A. Пустое поле:**
1. Оставьте поле поиска пустым.
2. Нажмите «Найти».
3. Программа покажет ошибку: «Поле поиска не должно быть пустым!».
4. Курсор автоматически переместится в поле ввода.

* **Сценарий B. Несуществующий пользователь:**
1. Введите заведомо несуществующий логин, например `nonexistentuser12345`.
2. Нажмите «Найти».
3. Появится предупреждение: «Пользователь 'nonexistentuser12345' не найден на GitHub».

* **Сценарий C. Повторное добавление в избранное:**
1. Найдите и добавьте в избранное пользователя `octocat`.
2. Снова выберите его в результатах поиска.
3. Нажмите «Добавить в избранное».
4. Программа покажет предупреждение: «Этот пользователь уже находится в списке избранных!».

**Пример 5. Проверка сохранения данных**

1. Добавьте 2–3 пользователей в избранное (например, `octocat`, `torvalds`, `defunkt`).
2. Закройте приложение (через крестик в углу окна).
3. Запустите его снова командой `python main.py`.
4. Откройте раздел «Избранные пользователи».
5. Ожидаемый результат: все ранее добавленные пользователи отображаются в списке.
6. Проверьте файл `users.json` — он должен содержать JSON‑массив с данными этих пользователей.

**Пример 6. Поиск с использованием разных регистров**

1. Введите имя пользователя в нижнем регистре: `octocat`.
2. Выполните поиск — приложение должно найти пользователя.
3. Введите то же имя в верхнем регистре: `OCTOCAT`.
4. Выполните поиск — GitHub API не чувствителен к регистру, поэтому пользователь будет найден.
5. Добавьте его в избранное.
6. Обратите внимание: приложение распознаёт дубликаты по полному тексту строки, поэтому `octocat` и `OCTOCAT` могут считаться разными записями.

**Пример 7. Проверка работы при проблемах с сетью**


* **Сценарий A. Отключённый интернет:**
1. Отключите интернет‑соединение (отсоедините кабель или выключите Wi‑Fi).
2. Попробуйте выполнить поиск любого пользователя.
3. Программа покажет ошибку: «Не удалось подключиться к GitHub API. Проверьте интернет‑соединение».
4. Подключите интернет и повторите поиск — он должен выполниться успешно.

* **Сценарий B. Временные проблемы с API:**
1. Если GitHub API временно недоступен, приложение покажет ошибку с кодом сервера.
2. Подождите несколько минут и повторите запрос.

**Пример 8. Проверка содержимого файла users.json**

1. Добавьте нескольких пользователей в избранное.
2. Закройте приложение.
3. Откройте файл `users.json` в любом текстовом редакторе.
4. Вы увидите структурированный JSON‑файл примерно такого вида:
* [
"octocat | The Octocat | Компания: GitHub | Локация: San Francisco | Репозиториев: 8",
"torvalds | Linus Torvalds | Компания: Linux Foundation | Локация: Portland, OR | Репозиториев: 15"
]

### Требования для запуска программы

**1. Системные требования:**
* Операционная система: Windows 7+, macOS 10.12+, Linux (с графической оболочкой)
* Python версии 3.6 или выше
* Доступ в интернет (для работы с GitHub API)

**2. Необходимые библиотеки Python:**
* `requests` — для выполнения HTTP‑запросов к GitHub API

  * `tkinter` (графический интерфейс)
  * `json` (работа с JSON)
  * `os` (работа с файловой системой)
